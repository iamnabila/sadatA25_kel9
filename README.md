Abstrak
Repositori digital digilib.uns.ac.id menyimpan ribuan skripsi mahasiswa Universitas Sebelas Maret, tetapi sistem pencariannya belum mendukung pengurutan hasil berdasarkan relevansi dan tahun terbaru, serta belum menyediakan fitur rekomendasi skripsi serupa. Penelitian ini membangun sistem pencarian skripsi yang menjawab kedua permasalahan tersebut menggunakan struktur data ArrayList, HashMap, dan Graph. Sistem menerapkan Inverted Index untuk pencarian cepat, Merge Sort untuk pengurutan hasil berdasarkan skor relevansi maupun tahun terbit, Jaccard Similarity untuk menghitung kemiripan antara abstrak, dan Breadth-First Search (BFS) untuk menampilkan rekomendasi skripsi serupa. Sistem juga dilengkapi dengan penanganan kata negasi dan perluasan sinonim agar hasil pencarian lebih akurat. Pengujian dilakukan pada 50 data skripsi dalam lingkup Fakultas Teknologi Informasi dan Sains Data. Hasil pengujian menunjukkan sistem mampu mengembalikan  hasil pencarian terurut, relevan, serta memberikan rekomendasi skripsi serupa dengan benar. 

Kesimpulan
Jawaban terhadap rumusan masalah
Sistem yang dibangun dapat menjawab dua permasalahan utama. Pertama, hasil pencarian kini diurutkan berdasarkan skor relevansi dan apabila skornya sama, diurutkan berdasarkan tahun terbit terbaru menggunakan Merge Sort. Kedua, sistem menyediakan fitur rekomendasi skripsi serupa berdasarkan kemiripan abstrak menggunakan Similarity Graph dan algoritma BFS, sehingga pengguna tidak perlu melakukan pencarian berulang untuk menemukan referensi yang berkaitan. 

Refleksi
Refleksi mengenai peran struktur data dalam efektivitas penyelesaian proyek
Proyek ini menunjukkan bahwa pemilihan struktur data yang tepat sangat menentukan efisiensi solusi. Penggunaan HashMap sebagai Inverted Index membuat pencarian jauh lebih cepat dibandingkan dengan pencarian linear. Graph terbukti tepat untuk merepresentasikan hubungan kemiripan antarskripsi yang bersifat banyak ke banyak, dan BFS (Breadth First Search) memungkinkan penelusuran skripsi serupa secara bertingkat. Merge Sort dipilih karena memberikan jaminan kompleksitas O(M log M) yang konsisten. Secara keseluruhan, kombinasi ArrayList, HashMap, dan Graph berhasil menyelesaikan permasalahan pencarian skripsi secara efektif dan efisien.
